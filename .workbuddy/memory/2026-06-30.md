### Critical learnings on validation (DURING SESSION)
- The number regex `\$?\d[\d,]*(?:\.\d+)?%?` matches digits in BOTH value and context
- **Numbers in value+context must all appear in transcript exactly** (or as substring, e.g. "517" not "5170")
- Common gotchas to avoid:
  - "1.5万" in Chinese context gets parsed as "1.5" — must use "$15" form when transcript says "15 20k"
  - "5170亿美元" extracted as "5170" — but transcript only has "517" — match exact digit sequence
  - "8万美元" extracted as "8" — transcript has "$80,000" but NOT "8" alone
  - "5个" in Chinese context gets parsed as "5" — use words like "数个" or check transcript
  - "1.8亿" extracted as "1.8" — use "$180 million" form to match transcript
  - "4月" extracted as "4" — transcript has "April" — use "April" in context
  - "2025年" extracted as "2025" — use "今年" instead
- **The two L's: `lstrip("$")` + `replace(",", "")`** in `numbers()` function
- Pattern that works: write `value` field using English form (e.g. "$180 million", "15 20k", "517 billion") that matches transcript verbatim, and put Chinese interpretation in `context` carefully avoiding stray digits
- **Quote must be ≤46 non-whitespace chars** (compact_text limit) AND must exist as substring in transcript with apostrophes preserved (e.g. "if you're" not "if you are")
- Verbatim quote fails if transcript splits word across line break (e.g. "hysteria would be a" + newline + "a good word" — pick shorter phrase)
